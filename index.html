2. gsap.to() and gsap.from()

The GSAP scripts you added in the previous step gives us access to the gsap object. This object includes a lot of methods, but let‚Äôs have a closer look at 2 of them : gsap.from() and gsap.to() .

I‚Äôll take as an exemple an empty HTML with just a blue square :

<style>
.square {
	width: 50px;
	height: 50px;
	background-color: blue;
}
</style>
<div class="square"></div>

Inside my script i‚Äôll call the gsap.to() method. 

gsap.to('.square', {
  opacity: 0,
  x: 400,
  duration: 1
})

Let me break down what i did :

  1. The first parameter of the method is the element i want to animate. I can select it like in the CSS, through its class (square in this case).
  1. The second parameter is an object with the value of every parameter i want to change. In this situation, I wan‚Äôt my element to become transparent and to move horizontally by 400 pixels to the right. I also indicated the duration of the animation in seconds.

GSAP figures out the current values automatically (you don't need to define starting values) and animate what you selected to make it go to the indicated values in the indicated duration.

Result :

The blue square goes from its default static state to being 400 pixels on the right and transparent, the whole animation taking 1 second, as indicated in the second argument of the function !

Recommandation in the use of gsap.to() and gsap.from() :

While you may be tempted to use gsap.from() to make your elements appear on your page, you might encounter issues like the element first appearing in its default state for a moment before going in its ‚Äúbefore-the-animation‚Äù state. This issue is due to your HTML and CSS loading before the scripts are applied hence showing the elements prematurely.

The way to solve this issue is to use gsap.to() . Lets take the blue square from earlier. Instead of using gsap.from() to make it appear like so :

gsap.from('.square', {
  opacity: 0,
  x: 100,
  duration: 1
})

I‚Äôll set in the CSS the default state of my square as transparent and 100 pixels on the right :

.square {
  width: 50px;
  height: 50px;
  background-color: blue;

  transform: translateX(100px);
  opacity:0;
}

Then in my gsap.to() function, i make it go back to its an opacity of 1 and its initial positioning :

gsap.to('.square', {
  opacity: 1,
  duration: 1,
  x: 0,
})

The result seems the same as using gsap.from() , but if a user as a bad connection or if your HTML has a lot of content, the elements will appear hidden even before the script is loaded, so you won‚Äôt encounter the issue i mentioned earlier.

You have now learned how to make animations. However, the animations you have set up fires before the user has scrolled to the concerned section ! Let‚Äôs see how you can use GSAP ScrollTrigger to control when to fire the animations !

3. ScrollTrigger

The first thing to do before everything is to indicate to GSAP that we have the ScrollTrigger script used in our site. To do that, add the following line at the start of your animation script :

gsap.registerPlugin(ScrollTrigger)

It is actually very easy to implement ScrollTrigger. Inside your animation, you can now indicate a ScrollTrigger property which will have for value an element, selected through CSS selectors just like the first argument of your animation function :

Let‚Äôs say I have 2 sections in my page which both takes the entire window and that in my second section which has a class of wrapper , i have my blue square from earlier. 

<section class="hero">
  <h2>First section</h2>
</section>
<section class="wrapper">
  <h2>Second section</h2>
  <div class="square"></div>
</section>
section {
  width: 100vw;
  height: 100vh;
}
.square {
  height: 100px;
  width: 100px;
  background-color: blue;
  
  opacity: 0;
  transform: translateX(-100px);
}

By adding my wrapper section as the ScrollTrigger for my square animation, I link my animation to the section so it only plays when the section is in the user viewport ! Let‚Äôs also add a delay to my animation to make it more obvious :

gsap.to('.square', {
  scrollTrigger: '.wrapper',
  opacity: 1,
  x: 0,
  duration: 1.5,
  delay: 1,
})

And the result is there. The animation only fires when the wrapper is in the user viewport !
